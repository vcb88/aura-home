# Пошаговое Руководство по Настройке Сервера для AURA

Это руководство проведет вас через все этапы настройки вашего мини-ПК с нуля до полностью работающего стека сервисов, необходимых для разработки и запуска проекта AURA.

---

### **Часть 1: Начальная Настройка Сервера**

На этом этапе мы установим операционную систему и подготовим ее к работе.

**Шаг 1.1: Установка Операционной Системы**
1.  Скачайте образ **Ubuntu Server 22.04 LTS** с официального сайта.
2.  Создайте загрузочную флешку.
3.  Установите систему на ваш мини-ПК, следуя инструкциям на экране.
4.  **Важно:** Во время установки обязательно отметьте галочкой пункт **"Install OpenSSH server"**. Это позволит вам подключаться к серверу удаленно с вашего основного компьютера.
5.  После установки и перезагрузки, найдите IP-адрес вашего сервера (например, в настройках вашего роутера) и подключитесь к нему по SSH:
    ```bash
    ssh ваш_логин@<IP-адрес_сервера>
    ```

**Шаг 1.2: Обновление Системы**
Первым делом после входа на сервер обновим все пакеты до последних версий.

```bash
sudo apt update && sudo apt upgrade -y
```

**Шаг 1.3: Установка Базовых Утилит**
Установим несколько полезных программ, которые пригодятся в работе.

```bash
sudo apt install -y git curl wget nano
```

---

### **Часть 2: Установка Docker и Docker Compose**

Мы будем запускать все наши сервисы в Docker-контейнерах. Это стандарт индустрии, который обеспечивает изоляцию и простоту управления.

**Шаг 2.1: Установка Docker Engine**
Самый простой способ — использовать официальный скрипт установки.

```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
```

**Шаг 2.2: Настройка Docker для работы без `sudo`**
Чтобы не писать `sudo` перед каждой командой `docker`, добавим вашего пользователя в группу `docker`.

```bash
# Замените $USER на ваш логин, если команда не сработает автоматически
sudo usermod -aG docker $USER
```
**Важно:** После выполнения этой команды вам нужно **перелогиниться** на сервер (выйти и зайти по SSH снова), чтобы изменения вступили в силу.

**Шаг 2.3: Проверка Установки Docker**
После переподключения выполните команду:
```bash
docker --version
```
Вы должны увидеть версию Docker, что подтверждает успешную установку.
